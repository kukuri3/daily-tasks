## 1. 目的

学科長業務の効率化のため、Outlookメールから未完了タスクを自動抽出するシステムを構築しています。現在はブラウザ操作でメールを読み取っていますが、安定性・セキュリティの観点からMicrosoft Graph APIによるプログラム的なアクセスに移行したいと考えています。

## 2. 申請内容

Azure Active Directory（Entra ID）にアプリケーションを1件登録し、以下の権限を付与いただきたくお願いいたします。

### アプリ登録情報

| 項目 | 値 |
|---|---|
| アプリ名 | Daily Task Extractor（任意） |
| アプリの種類 | パブリック クライアント（デスクトップアプリ） |
| リダイレクトURI | `http://localhost`（デバイスコードフロー使用） |
| サポートされるアカウントの種類 | この組織ディレクトリのみ |

### 必要なAPI権限

| 権限 | 種類 | 用途 |
|---|---|---|
| `Mail.Read` | 委任（Delegated） | 自分のメールの読み取り |

**注意事項：**
- `Mail.Read`は**委任（Delegated）権限**であり、本人のメールのみ読み取り可能です
- 他のユーザーのメールにはアクセスしません
- メールの送信・削除・変更は行いません（読み取り専用）
- アプリケーション権限（Application）は不要です

### 認証フロー

**デバイスコードフロー**を使用します。
1. プログラムがMicrosoftの認証URLとコードを表示
2. ユーザー（美馬）がブラウザでそのURLを開き、コードを入力してログイン
3. 以降はトークンが自動更新され、再ログイン不要

この方式はクライアントシークレットが不要で、ユーザー本人の明示的な認証が必要なため、セキュリティリスクが最も低いフローです。

## 3. 技術的な補足

### システム構成

```
[Graph API: Mail.Read]
    ↓ デバイスコードフロー認証
[ローカルPC上のPythonスクリプト]
    ↓ メール件名・本文を読み取り
[タスク抽出処理]
    ↓
[HTMLタスクリスト生成]
    ↓
[GitHub Pages で公開]
```

### データの取り扱い

| 項目 | 内容 |
|---|---|
| 取得するデータ | メール件名、差出人、受信日時、本文テキスト |
| 取得しないデータ | 添付ファイル、他ユーザーのメール |
| データの保存先 | ローカルPC内のJSONファイルのみ |
| 外部送信 | タスクの要約テキストのみGitHub Pagesに公開（メール本文は送信しない） |
| トークンの保存 | ローカルPCの暗号化されたキャッシュファイル（MSAL標準） |

### 使用ライブラリ
- MSAL（Microsoft Authentication Library）for Python
- Microsoft公式の認証ライブラリ

## 4. 登録手順（IT管理者向け）

Azure Portal（https://portal.azure.com）での操作手順です。

### 手順1: アプリ登録
1. Azure Portal → **Azure Active Directory** → **アプリの登録** → **新規登録**
2. 以下を入力:
   - 名前: `Daily Task Extractor`
   - サポートされるアカウントの種類: **この組織ディレクトリのみ**
   - リダイレクトURI: **パブリック クライアント** → `http://localhost`
3. **登録** をクリック

### 手順2: API権限の追加
1. 登録したアプリ → **APIのアクセス許可** → **アクセス許可の追加**
2. **Microsoft Graph** → **委任されたアクセス許可**
3. `Mail.Read` を検索して選択 → **アクセス許可の追加**
4. （オプション）**管理者の同意を与える** をクリック

### 手順3: デバイスコードフローの有効化
1. 登録したアプリ → **認証**
2. **詳細設定** → **パブリック クライアント フローを許可する** → **はい**
3. **保存**

### 手順4: 申請者への情報共有
以下の2つの値を申請者（美馬）にお知らせください：
- **アプリケーション（クライアント）ID**: アプリの概要ページに表示
- **ディレクトリ（テナント）ID**: アプリの概要ページに表示

## 5. セキュリティに関する補足

- 本アプリは**読み取り専用**（Mail.Read）であり、メールの送信・削除・変更は一切行いません
- **委任権限**のため、美馬本人のメールのみアクセス可能です
- デバイスコードフローは**クライアントシークレット不要**で、漏洩リスクがありません
- Microsoft公式のMSALライブラリを使用するため、トークン管理はMicrosoft標準のセキュリティに準拠します
- 必要に応じて、Azure Portal側からいつでもアプリを無効化・削除できます
